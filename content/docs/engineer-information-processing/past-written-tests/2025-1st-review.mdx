### 응집도

> 다음 중 자주 재사용되고, 유지보수가 용이한 응집도는 무엇인가?

<Callout>
  **자주 재사용되고 유지보수가 용이하다**는 것은 **응집도가 높은 것**을
  의미한다. 즉, **높은 응집도 유형**을 고르면 된다.
</Callout>

<br />

### 플랫폼 성능 특성 분석

> 소프트웨어 설계 시 구축된 플랫폼의 성능 특성 분석에 사용되지 않는 항목은?

<Callout title="플랫폼 성능 특성 분석 항목">
  - **응답시간**: 사용자 요청에 대해 **시스템이 응답하는 시간**
  - **가용성**: 시스템이 정상적으로 **운영될 수 있는 시간의 비용**
  - **사용률**: CPU, 메모리, 등 시스템 **자원의 사용 비율**
  - **경과시간**: 요구부터 **처리 완료까지의 시간**

</Callout>

<Callout type="error">
  <strong>'플랫폼'</strong>이었기 때문에, 처리량은 항목에 해당하지 않다고 해석
</Callout>

### 최소힙 정렬

> 힙에서 루트 노드를 제거 후, 트리의 구조를 골라라

<Callout>
  힙은 **하위 노드가 상위 노드보다 큰 값을 가지면 안된다**는 규칙을 갖는다. 또한
  첫 번째 인덱스는 비운다.
</Callout>

<Callout title='루트 노드가 제거된 뒤 정렬'>
  1. **마지막 인덱스 노드를 루트로 이동**시킨다.
  2. 자식 노드 중 **더 작은 노드와 비교하여 변경하는 정렬을 반복**한다.
  3. 마지막 노드와 비교가 끝나면 정렬을 중단한다.

</Callout>

### 최대힙

> 다음은 최대힙이다. 90을 삽입한 후의 최대힙은?

```
- 80 70 30 55 65 20 10 40 50 60
```

<Callout>
  최소힙과 반대로 **마지막에 삽입**을 한 뒤 **부모 노드와 비교하여 교체하는
  정렬을 반복**한다. 위 예시의 결과는 아래와 같다.

```
- 90 80 30 55 70 20 10 40 50 60 65
```

</Callout>

### SQL 명령어

> '회원' 테이블 생성 후 '주소' 필드(컬럼)가 누락되어 이를 추가하려고 한다. 이에 적합한 SQL 명령어는?

<Callout>
  **테이블 생성 후 필드 추가**는 DDL 명령어 중 `ALTER` 를 활용한다. `ALTER` 는
  **기존 테이블의 구조를 변경**할 때 사용한다.

```sql
ALTER TABLE 회원 ADD 주소 VARCHAR(255)
```

</Callout>

### SQL 구문

> 프로젝트 번호(PNO) 1, 2, 3에서 일하는 사원의 주민등록번호(JUNO)를 중복 없이 검색하는 SQL 문을 완성하시오.
>
> ```sql
> SELECT (   ) JUNO FROM WORKS WHERE PNO (   ) (1, 2, 3);
> ```

<Callout>
  **중복 없이 검색**하는 옵션은 `DISTINCT` 이다. 또한, `IN` 으로 **여러 값을
  포함하는 조건을 설정**할 수 있다.
</Callout>

<Callout title='기본키(Primary Key)'>

### IPv6

> 다음 중 IPv6에 대한 설명으로 틀린 것은?

<Callout title='IPv6'>

- **주소**는 **128비트**로 구성되어 있다.
- **주소**는 **16진수**로 표현되며, 연속된 0은 <strong>::</strong>로 축약할 수 있다.
- **헤더 크기**는 **40바이트**로 고정되어 있으며, **확장 헤더**를 통해 추가 기능을 제공한다.
- **유니캐스트, 애니캐스트, 멀티캐스트** 방식이 지원된다.

</Callout>

### 서브넷 마스크

> CIDR(Classless Inter-Domain Routing) 표기로 203.241.132.82/27과 같이 사용되었다면, 해당 주소의 서브넷 마스크는?

<Callout title="서브넷 마스크 계산">
`/27` 의 의미는 **서브넷 마스크 비트의 개수**를 의미한다. 아무런 명시가 없으면
**IPv4**에 해당하고 IPv4는 **32비트**이기에 **총 32비트의 구조에서 왼쪽부터
서브넷 마스크의 개수 만큼 1을 채우면 된다.**

**서브넷 마스크**는 `11111111.11111111.11111111.11100000` 이고 이것을 **10진수** 형태로 변환하면, `255.255.255.224` 에 해당한다.

</Callout>

### 메타버스

> 다음 중 미국 작가 "닐 스티븐슨"의 SF 소설 "스노우 크래시"에서 처음 등장한 개념으로, 현실 세계와 가상 세계가 합쳐진 개념을 의미하는 것은 무엇인가?

<Callout>
  해당 문제는 <strong>메타버스(Metaverse)</strong>에 대한 설명이다.

- **교육 분야**: **가상 교실과 캠퍼스**
- **산업 분야**: **BMW의 가상 공장**
- **의료 분야**: **원격 수술 교육 및 시뮬레이션**
- **기업 환경**: **가상 오피스**

</Callout>

### 쿠버네티스

> 다음 설명에 해당하는 신기술 용어는?
>
> 컨테이너화된 애플리케이션을 자동으로 배포, 확장, 관리하는 오픈소스 플랫폼

<Callout title='쿠버네티스(Kubernetes)'>
  **컨테이너** 기반 **서버 관리 자동화 도구**이다.

1. **자동 관리**: 프로그램이 꺼져도 자동 복구 가능
2. **손쉬운 확장**: 사용자가 많아지면 자동으로 서버 추가
3. **중단 없는 업데이트**: 서비스를 멈추지 않고 수정 가능

</Callout>
