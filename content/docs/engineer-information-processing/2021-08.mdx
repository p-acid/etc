### 27. 스택 알고리즘

> 다음은 스택의 자료 삭제 알고리즘이다. ⓐ에 들어 갈 내용으로 옳은 것은? (단, Top: 스택포인터, S: 스택의 이름)

<Callout>
  `If Top = 0` 구문은 스택 공간에 데이터가 없는 것을 의미하기에, **Underflow**가
  정답이다.
</Callout>

```
If Top = 0 Then
    (     ⓐ     )
Else {
    remove S(Top)
    Top = Top - 1
}
```

1. Overflow → 스택 공간이 가득 찼을 때 데이터를 추가하는 경우, 스택 오버플로우 발생 및 프로그램에 오류 야기
2. Top = Top + 1
3. Underflow → 스택 공간에 데이터가 없는데 프로그램에서 스택 데이터를 꺼내는 경우, 스택 언더플로우 발생 및 프로그램에 오류 야기
4. Top = Top → 스택이 비어있음을 명시

### 47. 관계 대수

> 관계 데이터베이스에 있어서 관계 대수 연산이 아닌 것은?

<Callout title="관계 대수(Relation Algebra)">
  관계 대수식이란 **기존 릴레이션(테이블)으로 부터 새로운 릴레이션을 생성**하는
  절차적 언어 문법이다. 추가로 관계 대수의 8대 연산자는 아래와 같다.

- **셀렉션(σ)**: 원하는 데이터를 **수평적으로 도출**
- **프로젝션(Π)**: 원하는 데이터를 **수직적으로 도출**
- **합집합(∪)**: **두 릴레이션의 튜플을 합치고**, 중복 튜플은 제외
- **교집합(∩)**: **두 릴레이션의 튜플들의 겹치는 부분**만을 도출
- **차집합(-)**: **앞 릴레이션에서 교집합을 제외한 부분**만을 도출
- **카티션 곱 연산자(×)**: 두 릴레이션에서 가능한 모든 조합을 생성
- **조인 연산자(⋈)**: **두 릴레이션으로부터 연관된 튜플**들을 결합
- **디비전(÷)**: 한 테이블에서 **다른 테이블의 모든 값을 가지고 있는 행**들을 찾음

</Callout>

1. 디비전(Division)
2. 프로젝트(Project)
3. 조인(Join)
4. ~~포크(Fork)~~

3. Coincidental Cohesion(우연적 응집도) → 모듈 내 구성요소들이 뚜렷한 관계 없이 묶인 경우
4. Sequential Cohesion(순차적 응집도) → 모듈 내 구성요소들이 이전의 명령어로부터 나온 출력 결과를 그 다음 명령어의 입력자료롤 사용하는 경우

### 62. 자동반복 요청방식

> 오류 제어에 사용되는 자동반복 요청방식(ARQ)이 아닌 것은?

<Callout title="자동반복 요청방식(ARQ)">
  오류 발생 시, **수신 측은 오류 발생을 송신 측에 통보**하고 **송신 측은 오류
  발생 븡록을 재전송하는 절차**를 의미한다.
</Callout>

1. Stop-and-wait ARQ → 송신 측에서 한 개의 블록을 전송한 후, 수신 측으로부터 응답을 기다리는 방식
2. Go-back-N ARQ → 연속 ARQ 중 하나로, 오류가 발생한 블록 이후의 모든 블록 재전송
3. Selective-Repeat ARQ → 연속 ARQ 중 하나로, 오류가 발생한 블록만을 재전송. 수신 측에서 순서대로 조합해야 함
4. ~~Non-Acknowledge ARQ~~

- Adaptive ARQ → 데이터 블록의 길이를 채널의 상태에 따라 동적으로 변경. 전송 효율이 제일 좋지만, 제어 회로가 복잡하고 비용이 많이 들어 거의 사용하지 않음

### 82. 접근통제 방법

> 정보 시스템 내에서 어떤 주체가 특정 개체에 접근하려 할 때 양쪽의 보안 레이블(Security Label)에 기초하여 높은 보안 수준을 요구하는 정보(객체)가 낮은 보안 수준의 주체에게 노출되지 않도록 하는 접근 제어 방법은?

1. Mandatory Access Control(강제 접근통제) → 권한 할당을 정책과 규칙에 따라 강제하는 모델
2. User Access Control(사용자 계정 컨트롤) → 프로그램에서 관리자 수준의 권한이 필요한 작업을 수행할 때, 사용자에게 알려 제어할 수 있도록 돕는 기능
3. Discretionary Access Control(임의 접근통제) → 사용자가 자원에 대한 접근 권한을 결정하는 모델
4. Data-Label Access Control(자료별 접근통제) → 개별 행, 열에 대한 쓰기 권한, 읽기 권한을 가졌는지 명확하게 결정하는 모델
